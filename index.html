Authentication.php

The following form accesses an API which will allow you to create a new session. It will return a JSON object with the username and password of the user which will be checked and verified within the API. If the credentials match, a new session will start else it will show an error message.
POST request is used to send the credentials to the API.
This could have also be done in a more structured manner where the list of users is stored in a database and upon succesful verification a session id and authentication key is generated and stored in the database becoming the token which is then sent to the client to grant access to it for future requests.

Additional Features!

Check Authentication Status
checkAuthenticationStatus.php - To check if you are authenticated, visit this API. You do not need to pass any additional parameters as the check is conducted using the sessions. If you are authenticated, you would get a success message, if not try authenticating to see a success response.
checkStudentName.php (you need to be logged in/authenticated) - To check the name of a student from the team, visit this API. You need to pass the student ID of one of the four team members as a parameter. If the student ID is valid, you will get the name of the student, if not, you will get an error message.
Logout
logout.php - Self explanatory. Destroys the session, effectively logging you out.
